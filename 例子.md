可以。下面我给你一个“从登录到命中”的完整具体例子。为了方便理解，我把真实密文简化写成 `DES(...)`、`RSA(...)`。

**场景**
- 玩家 A：`player_a`
- 玩家 B：`player_b`
- 房间：`A1B2C3`
- 游戏：双人对战
- 事件：A 开枪命中 B

---

## 1. A 先登录，拿到 Kerberos 票据

### Step 1: A -> AS
```json
{
  "type": "AS_REQ",
  "payload": {
    "username": "player_a",
    "password_cipher": "RSA(口令材料)",
    "nonce": "n1",
    "timestamp_ms": 1776650405000
  }
}
```

### Step 2: AS -> A
```json
{
  "type": "AS_REP",
  "payload": {
    "tgt": "DES(K_tgs, {user_id:10001, k_c_tgs:'abc123', expire_at:'12:00'})",
    "client_part": "DES(K_user, {k_c_tgs:'abc123', expire_at:'12:00', nonce:'n1'})"
  }
}
```

A 这时得到：
- `TGT`
- `K_c_tgs`

---

## 2. A 向 TGS 申请 Game Server 的票据

### Step 3: A -> TGS
```json
{
  "type": "TGS_REQ",
  "payload": {
    "service_name": "game-server-1",
    "tgt": "TGT_A",
    "authenticator": "DES(K_c_tgs, {user_id:10001, timestamp_ms:1776650410000, nonce:'n2'})"
  }
}
```

### Step 4: TGS -> A
```json
{
  "type": "TGS_REP",
  "payload": {
    "service_ticket": "DES(K_gs, {user_id:10001, k_c_gs:'gskeyA', expire_at:'11:05'})",
    "client_part": "DES(K_c_tgs, {k_c_gs:'gskeyA', expire_at:'11:05', nonce:'n2'})"
  }
}
```

A 这时得到：
- `Service Ticket`
- `K_c_gs`

---

## 3. A 进入 Game Server

### Step 5: A -> Game Server
```json
{
  "type": "GS_AUTH",
  "payload": {
    "service_ticket": "ServiceTicket_A",
    "authenticator": "DES(K_c_gs, {user_id:10001, timestamp_ms:1776650415000, nonce:'n3'})"
  }
}
```

### Step 6: Game Server -> A
```json
{
  "type": "GS_AUTH_OK",
  "payload": {
    "session_id": "sess-a-001",
    "user_id": 10001,
    "username": "player_a"
  }
}
```

B 也会走同样流程，拿到自己的 `session_id`。

---

## 4. A 建房，B 加房，双方准备

### Step 7: A 创建房间
```json
{
  "type": "ROOM_CREATE_REQ",
  "session_id": "sess-a-001",
  "payload": {
    "map_id": "arena_01"
  }
}
```

### Step 8: Server -> A
```json
{
  "type": "ROOM_STATE_PUSH",
  "room_id": "room-123",
  "payload": {
    "room_code": "A1B2C3",
    "status": "WAITING",
    "players": [
      {"user_id": 10001, "slot_no": 1, "is_ready": false}
    ]
  }
}
```

### Step 9: B 加入房间
```json
{
  "type": "ROOM_JOIN_REQ",
  "session_id": "sess-b-001",
  "payload": {
    "room_code": "A1B2C3"
  }
}
```

### Step 10: Server -> A 和 B
```json
{
  "type": "ROOM_STATE_PUSH",
  "room_id": "room-123",
  "payload": {
    "room_code": "A1B2C3",
    "status": "FULL",
    "players": [
      {"user_id": 10001, "slot_no": 1, "is_ready": false},
      {"user_id": 10002, "slot_no": 2, "is_ready": false}
    ]
  }
}
```

### Step 11: 双方点准备
```json
{
  "type": "ROOM_READY_REQ",
  "session_id": "sess-a-001",
  "room_id": "room-123",
  "payload": {"is_ready": true}
}
```

```json
{
  "type": "ROOM_READY_REQ",
  "session_id": "sess-b-001",
  "room_id": "room-123",
  "payload": {"is_ready": true}
}
```

### Step 12: Server -> A 和 B
```json
{
  "type": "ROOM_STATE_PUSH",
  "room_id": "room-123",
  "payload": {
    "status": "READY",
    "can_start": true
  }
}
```

## 5. 开局

### Step 13: A 点击开始
```json
{
  "type": "MATCH_START_REQ",
  "session_id": "sess-a-001",
  "room_id": "room-123",
  "payload": {}
}
```

### Step 14: Server -> A 和 B
```json
{
  "type": "MATCH_COUNTDOWN_PUSH",
  "room_id": "room-123",
  "payload": {
    "match_id": "match-888",
    "phase": "COUNTDOWN",
    "countdown_ms": 3000
  }
}
```

3 秒后：

```json
{
  "type": "MATCH_EVENT_PUSH",
  "room_id": "room-123",
  "payload": {
    "event_type": "MATCH_START"
  }
}
```

---

## 6. 对战中：A 开枪，B 被命中

### Step 15: A 上传输入
A 在这一帧按了攻击键：

```json
{
  "type": "PLAYER_INPUT_CMD",
  "session_id": "sess-a-001",
  "room_id": "room-123",
  "payload": {
    "input_seq": 101,
    "move_x": 1,
    "move_y": 0,
    "jump_pressed": false,
    "dash_pressed": false,
    "attack_pressed": true,
    "interact_pressed": false,
    "aim_x": 0.92,
    "aim_y": 0.38
  }
}
```

### Step 16: B 上传输入
```json
{
  "type": "PLAYER_INPUT_CMD",
  "session_id": "sess-b-001",
  "room_id": "room-123",
  "payload": {
    "input_seq": 88,
    "move_x": -1,
    "move_y": 0,
    "jump_pressed": false,
    "dash_pressed": false,
    "attack_pressed": false,
    "interact_pressed": false,
    "aim_x": -1.0,
    "aim_y": 0.0
  }
}
```

### Step 17: Server 内部处理
Server 做这些事：
1. 取 A/B 最新输入
2. 推进角色状态
3. 判断 A 此帧触发攻击
4. 生成子弹/攻击判定
5. 判断 B 被命中
6. 计算：
   - B 百分比：`92.0 -> 104.5`
   - 击飞方向和速度
7. 记录事件

### Step 18: Server 先发事件
```json
{
  "type": "MATCH_EVENT_PUSH",
  "room_id": "room-123",
  "payload": {
    "event_type": "PLAYER_HIT",
    "event_seq": 45,
    "data": {
      "attacker_slot": 1,
      "target_slot": 2,
      "damage_add": 12.5,
      "target_damage_percentage": 104.5
    }
  }
}
```

客户端收到这个后，可以：
- 播放命中特效
- 播放受击音效
- 触发 UI 抖动

### Step 19: Server 再发最新快照
```json
{
  "type": "MATCH_SNAPSHOT_PUSH",
  "room_id": "room-123",
  "payload": {
    "match_id": "match-888",
    "phase": "PLAYING",
    "server_frame": 240,
    "players": [
      {
        "slot_no": 1,
        "x": 12.45,
        "y": 3.20,
        "state": "attack",
        "damage_percentage": 48.5,
        "stocks_left": 3
      },
      {
        "slot_no": 2,
        "x": 20.10,
        "y": 5.60,
        "state": "hit_fly",
        "damage_percentage": 104.5,
        "stocks_left": 2
      }
    ],
    "projectiles": [
      {
        "projectile_id": "proj-1",
        "owner_slot": 1,
        "x": 14.00,
        "y": 3.50
      }
    ]
  }
}
```

客户端收到这个后，在 Unity 里做：
- 更新玩家坐标
- 更新朝向/动画状态
- 更新百分比 UI
- 更新命数 UI
- 更新子弹位置

---

## 7. 如果 B 这时断线

Server 不会直接结束比赛，而是：

- 把 B 标记为 `is_online = false`
- B 后续输入按空输入处理：
```json
{
  "move_x": 0,
  "move_y": 0,
  "jump_pressed": false,
  "dash_pressed": false,
  "attack_pressed": false,
  "interact_pressed": false
}
```

然后继续发快照。  
所以你们可以理解成：

- **事件**告诉客户端：“刚刚发生了什么”
- **快照**告诉客户端：“现在世界是什么样”

---

## 一句话总结

一场完整交互不是：

- Client 发攻击
- Server 只回一个总快照

而是：

1. Client 发输入  
2. Server 做权威计算  
3. Server 发事件  
4. Server 发快照  
5. Client 用事件驱动特效/UI，用快照驱动场景状态
